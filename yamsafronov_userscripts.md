## [ Сценарий 1 – Регистрация и настройка аллергенов ]

1. Пользователь нажимает "Регистрация" на стартовом экране.

2. Система отображает форму: Имя, Email, Пароль.

3. После ввода данных и нажатия "Зарегистрироваться" происходит проверка данных на корректность. Система создает аккаунт и сразу предлагает выбрать "опасные" ингредиенты.

4. Пользователь отмечает в списке свои аллергены (например: лактоза, арахис, глютен).

5. Система сохраняет профиль на сервере и перенаправляет на экран сканера.


## [ Сценарий 2 – Вход в систему ]

1. Пользователь вводит Email и Пароль.

2. Система проверяет данные на сервере.

3. При успешной проверке токен сохраняется в памяти устройства, и пользователь попадает на главный экран.

4. Если данные неверны, выводится ошибка "Неверный email или пароль".


## [ Сценарий 3 – Сканирование штрих-кода (Успех) ]

1. Пользователь нажимает на иконку камеры на главном экране и наводит камеру на штрих-код продукта.

2. Система распознает код и отправляет запрос на сервер.

3. Сервер находит продукт в базе и сопоставляет его состав со списком аллергенов пользователя.

4. Система отображает карточку продукта с вердиктом: "Безопасно" (зеленый цвет) или "Опасно: содержит [1], [2]..." (красный цвет).



## [ Сценарий 4 – Товар отсутствует в базе ]

1. Пользователь сканирует штрих-код, которого нет в базе данных.

2. Система выводит сообщение: "Продукт не найден. Хотите добавить его?" и кнопку "Добавить вручную".

3. Пользователь соглашается, открывается форма добавления (см. Сценарий 5).



## [ Сценарий 5 – Ручное добавление продукта ]

1. Пользователь заполняет форму: Название, Бренд, Состав (текстом), Калорийность и БЖУ.

2. Пользователь нажимает "Сохранить".

3. Система отправляет данные на сервер, продукт привязывается к отсканированному ранее штрих-коду.

4. Теперь этот продукт доступен для проверки всем пользователям системы.



## [ Сценарий 6 – Редактирование списка аллергенов ]

1. Пользователь переходит в раздел "Профиль" -> "Мои аллергены".

2. Система отображает текущий список (например, только "Орехи").

3. Пользователь добавляет "Молочные продукты" и нажимает "Обновить".

4. Система синхронизирует данные с сервером. Теперь, например, при сканировании шоколада система будет предупреждать и о молоке.



## [ Сценарий 7 – Ведение дневника питания ]

1. После сканирования продукта пользователь нажимает кнопку "Добавить в дневник".

2. Система предлагает выбрать прием пищи (Завтрак/Обед/Ужин) и вес порции.

3. Данные сохраняются в дневнике питания на сервере.

4. В разделе "Дневник" пользователь видит список съеденного за день и может посмотреть питание за предыдущие дни.

5. Система отображает статистику: сколько калорий, белков, жиров и углеводов потреблено сегодня.



## [ Сценарий 8 – Просмотр истории проверок ]

1. Пользователь переходит в раздел "История".

2. Система отображает список последних 100 отсканированных товаров с мини-иконками безопасности.

3. При нажатии на элемент истории открывается полная карточка продукта с составом.



## [ Сценарий 9 – Просмотр и редактирование профиля ]

1. Пользователь переходит в раздел "Профиль".

2. Система отображает текущие данные: имя, email и список аллергенов.

3. Пользователь нажимает "Редактировать", изменяет имя пользователя и сохраняет.

4. Система отправляет обновлённые данные на сервер и подтверждает сохранение.



## [ Сценарий 10 – Установка дневного лимита калорий ]

1. Пользователь переходит в раздел "Профиль" → "Настройки питания".

2. Система отображает текущий лимит (по умолчанию 2000 ккал).

3. Пользователь вводит новое значение и нажимает "Сохранить".

4. Система сохраняет лимит на сервере. Теперь в разделе "Дневник" прогресс отображается относительно нового значения.
